# Хакатон Axenix

Репозиторий содержит документацию для сервиса, написанную в рамках хакатона Axenix, трек «Промпт-инжиниринг».

Команда "It Depends".

Участники:
* Васин Фёдор
* Галеева Диана
* Зашкола Алина
* Кертиева Сабина
* Скоморохов Александр

## Контекст задачи

С помощью промптинга разработать системный дизайн сервиса по генерации презентаций с учётом бизнес, функциональных и технических требований.

## Выбор LLM

В рамках решения задачи мы сравнили модели ChatGPT 5.2, Mistral, Claude, DeepSeek и Gemini.
Цель сравнения: выбрать LLM, в которую команда будет отправлять промпты для подготовки системного дизайна сервиса по генерации презентаций.

Важно: это выбор модели для этапа **аналитики и проектирования**, а не выбор LLM, которая будет работать внутри конечного продукта в production.

Для подготовки документации и системного дизайна выбрана модель **Claude**.

## Почему выбрали Claude для документирования решения

- Лучше сохраняет структуру длинных материалов: от бизнес-требований до технической архитектуры.
- Стабильно следует сложным инструкциям к формату (разделы, таблицы, матрицы трассировки, критерии приемки, готовые шаблоны).
- Дает более связные и логически непротиворечивые артефакты по архитектуре и функциональным сценариям.
- Удобно поддерживает итеративный режим работы: уточнение требований, ревизии, детализация по шагам.
- Качество текста подходит для быстрой передачи в работу аналитикам, архитекторам и разработке.

## Какие артефакты формируем через выбранную LLM

- Vision и scope решения.
- Бизнес-требования и ограничения.
- Функциональные и нефункциональные требования.
- Системный дизайн сервиса генерации презентаций.
- Описание модулей, интеграций и API-контрактов.
- Пользовательские сценарии, user stories, критерии приемки.
- Риски, допущения, roadmap и план валидации.

## Отдельно про выбор LLM в самом продукте

Выбор LLM для сервиса генерации презентаций выполняется отдельным этапом на основе иных критериев (качество генерации слайдов, latency, стоимость, надежность, безопасность, масштабируемость).
